# Sales Analytics System

> End-to-end data pipeline — from raw CRM exports to a fully reactive dashboard with real data cleaning, computed business insights, linear regression forecasting, and dynamic filters.

**[Live Demo →](https://bhaskar21-7.github.io/sales-analytics)**

## Overview

Sales teams often work with CRM exports riddled with duplicates, missing values, inconsistent formats, and type errors. This project builds a complete analytics system that handles every stage of that problem — from raw data ingestion to an interactive decision-support dashboard.

```
Raw CRM Export (3,328 records, injected quality issues)
        ↓
Python Data Cleaning Pipeline
(duplicates, nulls, type errors, casing standardisation)
        ↓
Feature Engineering + Exploratory Analysis
(KPIs, win rates, revenue concentration, AOV trends)
        ↓
Linear Regression Forecast (scikit-learn)
        ↓
React Dashboard
(state-driven, filter-reactive, real-time recomputation)
```

## Architecture

The React app follows a strict unidirectional data flow:

```
deals.json → App (useState) → filterDeals() → each Component
                ↑
           Filters.jsx (setCategory / setRep)
```

All data logic lives in `src/utils/compute.js` as pure functions — no DOM knowledge, no React imports. This makes every computation independently testable:

```js
// Pure function — takes data, returns data, no side effects
export function computeKPIs(deals) {
  const won     = deals.filter(d => d.status === 'Won');
  const revenue = won.reduce((s, d) => s + d.value, 0);
  const winrate = won.length / (won.length + lost.length) * 100;
  return { revenue, orders: won.length, winrate, aov: revenue / won.length };
}
```

Components receive only the data they need as props. Filters update state in `App.jsx`, which re-renders only what changed.

## Data Engineering

The notebook (`sales_analysis.ipynb`) handles:

- **Duplicates** — 128 records removed (~4% of raw dataset)
- **Null imputation** — 279 missing values across `rep`, `category`, `value` columns
- **Type errors** — 50 malformed value strings fixed (`"2300.0$"` → `float64`)
- **Casing standardisation** — `status` field inconsistencies resolved
- **Feature engineering** — `month`, `quarter`, `month_name` columns derived from `close_date`

All cleaning stats are surfaced in the Data Quality Audit panel on the dashboard.

---

## Forecasting

A linear regression model (scikit-learn) is trained on the 12 monthly revenue data points and projects the next 3 months. The model and its R² score are exported to `forecast.json` and displayed transparently in the dashboard as clearly labelled forecast bars.

This is documented as a **baseline model**, not a production forecasting system — which is the honest framing.

## How to Run

**Option 1 — Open the static build directly**

Download `sales_analytics_react_build.html` and open it in any browser. No server needed.

**Option 2 — Run the React dev server**

```bash
# Clone the repo
git clone https://github.com/YOURUSERNAME/sales-analytics.git
cd sales-analytics

# Install dependencies
npm install

# Start dev server
npm run dev

# Open http://localhost:5173
```

**Option 3 — Regenerate data from the notebook**

```bash
pip install jupyter pandas numpy matplotlib seaborn scikit-learn

jupyter notebook notebook/sales_analysis.ipynb
# Run all cells to regenerate deals.json and forecast.json
```
